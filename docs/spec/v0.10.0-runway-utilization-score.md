## Leitentscheidungen

### LE1 — Numerator: TD-Punkt-bis-Stop

```
distance_used_m = td_distance_from_threshold_m + rollout_distance_m
```

**Sowohl `td_distance_from_threshold_m` als auch `rollout_distance_m` sind hard-required.** Wenn eins fehlt → Skip (siehe LE6).

**R2-P0-1-Fix:** R2 hatte `unwrap_or(0.0)` — das hätte missing data versteckt und falsche 100 PTS produziert. Entfernt. **Kein Fallback auf rollout-only.**

`landing_float_distance_m` bleibt als reine Info-Zeile in der UI, nicht im Score (TD-Distanz enthält Float schon implizit über die Forensik-Definition).

### LE2 — Denominator: Landing Distance Available (LDA)

```
displaced_m = runway_displaced_threshold_ft.unwrap_or(0) as f32 * 0.3048
LDA_m = runway_length_m - displaced_m
```

Wenn `LDA_m <= 0` → Skip (`invalid_lda`). Datenfehler-Schutz.

### LE3 — Sonderfall `pre_displaced_threshold = true` → **Score-Cap auf 55 PTS**

**R2-P1-4-Fix:** R2 ließ den Score ungekappt mit reiner Warning-Pill. Thomas-Vorschlag: Cap auf 55 PTS (= „ok_stop"-Niveau max).

```rust
if input.pre_displaced_threshold.unwrap_or(false) {
    points = points.min(55);
    warning = Some("Vor Displaced-Threshold aufgesetzt — Anflugprofil prüfen");
}
```

Begründung: Pilot setzt VOR der nutzbaren Schwelle auf (= regelwidrig, Wing-Tip-Vortex- + Lärmschutz-Verstoß). Würde der Bremsweg „excellent" sein, wäre der Eindruck „alles super" — falsch. Mit Cap 55 sieht Pilot: „solides Stopping, aber: KEIN Excellent möglich wegen Aufsetz-Punkt".

`distance_used_m` Negativ-Schutz (R2 LE3 bleibt): TD vor Schwelle = negative `td_distance_from_threshold_m`, summiert auf einen Wert < `rollout_distance_m`. Wir clampen:

```rust
let raw_used = td_dist + rollout;
let distance_used = raw_used.max(rollout); // niemals < rollout
```

### LE4 — Score-Bänder, exakt definiert (R2-P2-1 Fix: Float-Banding)

```rust
// 1. Raw-Ratio als Float
let raw_ratio_pct: f32 = (distance_used / LDA_m) * 100.0;

// 2. Overrun-Check ZUERST auf RAW (vor Allowance!) — R2-P0-2-Fix
if raw_ratio_pct > 100.0 {
    return (0, "overrun_risk", Band::Bad);
}

// 3. Heavy-Allowance auf Float
let allowance_pp: f32 = if is_heavy(aircraft_icao) { 5.0 } else { 0.0 };
let effective_pct: f32 = raw_ratio_pct - allowance_pp;

// 4. Banding auf Float — keine Rundung vorher!
let (points, rationale, band) = match effective_pct {
    e if e < 30.0  => (100, "excellent_margin", Band::Good),
    e if e < 50.0  => ( 80, "good_stop",        Band::Good),
    e if e < 70.0  => ( 55, "ok_stop",          Band::Ok),
    e if e < 90.0  => ( 25, "long_rollout",     Band::Bad),
    _              => (  5, "marginal_runway",  Band::Bad),  // 90 <= effective <= 100
};

// 5. Rundung NUR fürs Display
let display_pct = raw_ratio_pct.round() as i32;
```

**Konsequenz:**
- `29.5%` Raw + Heavy-Allowance → 24.5% effective → `< 30` → `excellent_margin` ✓ (R2-P2-1-Fix: keine Pre-Rundung mehr)
- `103%` Raw → trotz Heavy-Allowance → `overrun_risk` (raw > 100, vor Allowance gechecked) ✓ (R2-P0-2-Fix)
- `100.0%` exakt → fällt in 4. Branch (`marginal_runway`, 5 PTS) weil `100 > 100` false und nicht in den anderen Branches matched

**Display-Format** zeigt das gerundete `display_pct` (= UI-Konsistenz), aber Score wird auf Float entschieden.

### LE5 — Heavy-Allowance, **als Reduktion vor Banding**

(R2 LE5 unverändert, lediglich Formel-Reihenfolge präzisiert in LE4)

```
allowance_pp = 5.0  if is_heavy else 0.0
effective_pct = raw_ratio_pct - allowance_pp
```

Heavy-Set bleibt identisch zur heutigen `landing-scoring::category_for_icao`-Liste (Wide-Bodies + B747 + MD11 + IL96 + Antonov). Light/Medium ohne Allowance.

**Q-R2-2 Antwort:** Allowance bleibt bei **5 pp**, nicht 10. 10 wäre fast ein ganzes Band → zu gnädig.

### LE6 — Skip-Gate erweitert (R2-P0-1, P1-2 Fixes)

```rust
fn skip_reason(input: &RolloutInput) -> Option<&'static str> {
    if input.rollout_distance_m.is_none() {
        return Some("missing_rollout_distance");  // NEU R3
    }
    if input.td_distance_from_threshold_m.is_none() {
        return Some("missing_td_distance");       // NEU R3
    }
    if input.runway_length_m.is_none() {
        return Some("missing_length");
    }
    // R2-P1-2-Fix: None ist NICHT trusted
    if input.runway_geometry_trusted != Some(true) {
        return Some("untrusted_geometry");
    }
    if input.airport_source.as_deref() != Some("runway_match") {
        return Some("off_airport_landing");
    }
    // LE2 LDA-Sanity
    let displaced_m = input.runway_displaced_threshold_ft.unwrap_or(0) as f32 * 0.3048;
    let lda = input.runway_length_m.unwrap() - displaced_m;
    if lda <= 0.0 {
        return Some("invalid_lda");
    }
    None
}
```

5 Skip-Reasons im Wire-Format dokumentiert. Kein Meter-Backup-Fallback — lieber „kein Score" als irreführend (R2-Antwort 6).

## Algorithmus (Rust-Implementierung, R3-Final)

```rust
pub struct RolloutInput<'a> {
    pub td_distance_from_threshold_m: Option<f64>,
    pub rollout_distance_m: Option<f32>,
    pub landing_float_distance_m: Option<f32>,
    pub runway_length_m: Option<f32>,
    pub runway_displaced_threshold_ft: Option<i32>,
    pub pre_displaced_threshold: Option<bool>,
    pub runway_geometry_trusted: Option<bool>,
    pub airport_source: Option<&'a str>,
    pub runway_match_icao: Option<&'a str>,
    pub runway_match_ident: Option<&'a str>,
    pub aircraft_icao: Option<&'a str>,
}

pub fn sub_rollout_v2(input: &RolloutInput) -> SubScoreEntry {
    // ── Skip-Gate (LE6) ──────────────────────────────────────────
    // R4-P1-Fix: SubScoreEntry::skipped hat 3 Args, nicht 2:
    //   pub fn skipped(key: &str, label_key: &str, reason: &str) -> Self
    if let Some(reason) = skip_reason(input) {
        return SubScoreEntry::skipped("rollout", "landing.sub.rollout", reason);
    }

    // Unwraps safe weil Skip-Gate alle None-Fälle gefiltert hat
    let td_dist  = input.td_distance_from_threshold_m.unwrap() as f32;
    let rollout  = input.rollout_distance_m.unwrap();
    let runway_m = input.runway_length_m.unwrap();
    let displaced_m = input.runway_displaced_threshold_ft.unwrap_or(0) as f32 * 0.3048;
    let lda_m = runway_m - displaced_m;

    // ── Numerator + Negativ-Schutz (LE1 + LE3 Halb-1) ────────────
    let raw_used = td_dist + rollout;
    let distance_used = raw_used.max(rollout); // niemals < rollout

    // ── Raw-Ratio als Float ──────────────────────────────────────
    let raw_ratio_pct: f32 = (distance_used / lda_m) * 100.0;

    // ── Overrun-Check ZUERST auf RAW (LE4 Step 2, R2-P0-2) ───────
    let (points, rationale, band) = if raw_ratio_pct > 100.0 {
        (0u8, "overrun_risk", Band::Bad)
    } else {
        // Heavy-Allowance auf Float (R2-P2-1: keine Pre-Rundung)
        let allowance_pp: f32 = if is_heavy(input.aircraft_icao) { 5.0 } else { 0.0 };
        let effective_pct: f32 = raw_ratio_pct - allowance_pp;

        match effective_pct {
            e if e < 30.0 => (100, "excellent_margin", Band::Good),
            e if e < 50.0 => ( 80, "good_stop",        Band::Good),
            e if e < 70.0 => ( 55, "ok_stop",          Band::Ok),
            e if e < 90.0 => ( 25, "long_rollout",     Band::Bad),
            _             => (  5, "marginal_runway",  Band::Bad),
        }
    };

    // ── pre_displaced-Cap (LE3 Halb-2, R2-P1-4) ──────────────────
    // R4-P1-Fix: bei Cap wird Rationale-Key auf "pre_displaced_capped"
    // ueberschrieben — sonst sieht UI "Viel Bahn-Reserve" bei 55 PTS,
    // ist unehrlich. Antwort R3-Q3: Override.
    let pre_displaced = input.pre_displaced_threshold.unwrap_or(false);
    let (capped_points, final_rationale, final_band) = if pre_displaced {
        let pts = points.min(55);
        // Band entsprechend der gecappten Punkte neu setzen
        let band = if pts >= 75 { Band::Good }
                   else if pts >= 45 { Band::Ok }
                   else { Band::Bad };
        (pts, "pre_displaced_capped", band)
    } else {
        (points, rationale, band)
    };

    let warning = if pre_displaced {
        Some("pre_displaced_threshold".to_string())  // i18n-Key fuer Warning-Pill
    } else { None };

    // ── Display-Strings (LE9) ────────────────────────────────────
    let display_pct = raw_ratio_pct.round() as i32;
    let value = format!("{} m / {} m  ·  {} %",
                        distance_used.round() as i32,
                        lda_m.round() as i32,
                        display_pct);

    let mut extra: Vec<String> = Vec::new();
    if let Some(float) = input.landing_float_distance_m {
        extra.push(format!("davon ~{} m Float vor Aufsetzen", float.round() as i32));
    }
    if let (Some(icao), Some(ident)) = (input.runway_match_icao, input.runway_match_ident) {
        extra.push(format!("Bahn: {} {}, LDA {} m", icao, ident, lda_m.round() as i32));
    }

    // ── Resultat via scored-Constructor (R4-P1-Fix) ──────────────
    // SubScoreEntry::scored(key, label_key, score, value, rationale, band)
    // - value ist String (nicht Option), wird im Constructor zu Some(...) gewrappt
    // - rationale ist &str, wird zu rationale_key="landing.rat.<rationale>"
    SubScoreEntry::scored(
        "rollout",
        "landing.sub.rollout",
        capped_points,
        value,
        final_rationale,
        final_band,
    )
    .with_extra(extra)        // NEU v0.10.0
    .with_warning(warning)    // existierendes Builder-Pattern auf v1
}
```

TypeScript-Render-Code in beiden TS-Files: **kein Compute**, nur Render aus `payload.sub_scores[].{value, points, rationale_key, band, extra, warning, skipped, reason}`.
