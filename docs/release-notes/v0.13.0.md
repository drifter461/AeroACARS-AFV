## 🇩🇪 Deutsch

**v0.13.0 — Großes Integritäts-Release: VA-Owner sieht ab sofort welche PIREPs vertrauenswürdig sind und welche durch Sim-Glitches, Slew-Mode, Sim-Pausen oder Telemetrie-Lücken kompromittiert wurden. Komplette Pipeline VPS-Recorder ↔ phpVMS-Modul. Für Piloten ändert sich UI-seitig wenig — der Validator läuft serverseitig und gibt sauberen Flügen freie Bahn.**

### Was ist neu

#### 🛡 Mid-Session-Integrity-Validator (Stream A–G, 145+ Test-Cases)

Der VPS-Recorder validiert jeden Position-Snapshot live gegen 12+ Detektoren:

- **Stream A — Continuous-Checks:** Position-Δ (phase-aware, Cruise/Taxi getrennt), Altitude-Δ, Fuel-Rate, Weight-Konsistenz, Ground-Elevation-Check, Air-to-Ground-Teleport, SIM_STATE_RESET-Detection (bekannte A321/B737-Default-Fuel-Werte)
- **Stream B — Gap-Handler:** Telemetry-Gaps (15s/60s/300s gestaffelt), Phase-Continuity nach Reconnect, Fuel-Telemetry-Unavailable, Aircraft-Identity-Shift (Variant/Family/Manufacturer)
- **Stream C — Score-Trust:** Hard-Triggers am Touchdown (GS=0, IAS<30kt, Parking-Brake an, Engines off, Sample quarantined, CRITICAL-Flag im [TD−60s; TD+30s]-Fenster) → `score_trust_level` mit drei Stufen: `trusted` / `review` / `untrusted`
- **Live-Aggregation:** Combo-Eskalationen (z.B. FUEL_TELEMETRY_UNAVAILABLE + FUEL_RATE_IMPOSSIBLE → CRITICAL)

#### 🚪 phpVMS Integrity-Gate-Modul (separates Repo)

Komplett neues Modul `MANFahrer-GF/aeroacars-integrity-gate` (Stream G — Drop-in ZIP wie DisposableBasic):

- `PirepFiled`-Listener hängt sich in den phpVMS-Submit-Flow ein, holt vom Recorder ein HMAC-signiertes Verdict
- Bei `verdict=clean` → ACCEPTED, bei `review/untrusted` → PENDING-Hold mit Status `INT_HOLD_*`
- **Auto-Repair-Service** patcht `auto_approve_acars=true`-Drift auf allen Ranks automatisch beim Boot + täglich per Cron + per-PIREP (Defense-in-Depth)
- Admin-UI in phpVMS: Settings + Review-Queue + Connection-Test
- Webhook für Admin-Decisions vom Recorder mit atomic decision-claim state-machine
- Discord-Alarme bei AutoRepair-Drift + Emergency-Reject

#### 📡 Client: MQTT Integrity-Flag-Subscriber

Der Pilot-Client abonniert ab v0.13.0 das neue MQTT-Topic `aeroacars/<va>/<pilot>/integrity_flag` und zeigt einen LIVE-Banner im Cockpit wenn der Recorder einen Anomalie-Flag während des Flugs erkennt:

- **Anomaly-Flag** (orange, dismissable): „Datenanomalie X in Phase Y"
- **Critical-Flag** (rot, blockierend, nicht-dismissable): „Der PIREP wird wahrscheinlich als 'untrusted' eingestuft"

#### 🔍 VPS-Side: 5 neue REST-Endpoints

- `GET /api/integrity-check/:pirep_id` — HMAC-verified Verdict-Endpoint für Gate
- `POST /api/integrity-check/:pirep_id/gate-ack` — Gate→Recorder Sync-Back (Two-Phase-Commit)
- `GET /api/pireps/:id/integrity` — PIREP-Detail mit Flags + Reasons
- `GET /api/admin/review-queue` — Liste aller PIREPs mit `requires_review=1`
- `POST /api/admin/pireps/:id/review` — Admin-Decision via Webapp (LE21b State-Machine)
- `POST /api/admin/integrity/backfill` — Dry-Run + Apply für historische PIREPs

### Was Piloten tun müssen

**Nichts.** Auto-Updater rollt v0.13.0 aus, beim Start ist der Integrity-Banner-Hook stumm bis ein Flag erscheint. Sauberen Flügen passiert nichts neu — sie gehen wie bisher als ACCEPTED durch.

Pilot-spürbare Änderungen:
- **Wenn Slew-Mode AN während Cruise:** Banner „Position-Δ-Anomalie" + PIREP wird als untrusted markiert
- **Sim-Pause + Fuel-Reset:** Banner „SIM_STATE_RESET" — PIREP geht in Review
- **MQTT-Reconnect > 5min während Cruise:** Banner „Telemetry-Gap" (anomaly, dismissable)
