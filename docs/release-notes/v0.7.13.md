## 🇬🇧 English

**Codebase audit + security cleanup — no more hardcoded token, ~700 LOC dead code removed, strict CSP.**

### What happened

Full QS audit of the pilot client with 3 parallel auditor agents. This release implements all pilot-client findings (VPS/SSH changes follow in a later release).

### Important for VA owners / pilot action needed

**🔴 Discord webhook is no longer hardcoded.** After updating to v0.7.13, **NO** Discord posts go out until you enter the URL once in Settings:

1. In your Discord server: delete the old GSG webhook → create a new one (Channel settings → Integrations → Webhooks)
2. Copy the URL
3. In AeroACARS: **Settings** → Discord Integration → paste the webhook URL

Reason: the token used to sit readable in the public GitHub repo — anyone could have spammed. Now the URL lives locally per pilot (or empty = no posts).

### Cleanup (pilots don't notice, code is healthy)

- Discord Rich Presence block (170 LOC, dead since v0.4.0 with a "wiring lands in v0.4.5" promise) → gone
- 4 orphan React components, 6 orphan Tauri commands, 8 Fenix FCU fields, several cargo-warned functions → gone
- `keyring` Rust dep (v0.5.15 migration, 30+ releases ago) + `schemars` workspace dep + `discord-rich-presence` dep → gone
- 4 dead i18n keys + entire dashboard locale block (DE/EN/IT) → gone
- 8 implemented/released specs archived to `docs/spec/historical/`

### Security hardening

- Tauri updater private key moved out of the repo working tree
- Tauri webview Content Security Policy from `null` to strict (default-src 'self', whitelisted connect-src for phpVMS / MQTT WS / SimBrief). XSS defense-in-depth

### What's next (v0.7.14 or later)

VPS-side patches: NOPASSWD:ALL → sudoers whitelist, rate limits on auth endpoints, `@fastify/static` major bump (CVE fix), Caddy security headers. Need an SSH test window.

Also: 4 vulnerabilities in `rustls-webpki` via `rumqttc` (transitive dep) — fix requires `rumqttc` major bump to `rustls 0.23+`.
