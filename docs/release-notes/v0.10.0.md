### Was ist neu

#### 🛬 Bahn-Auslastung neu (Runway-Utilization-Score)

Der Sub-Score „Bahn-Auslastung" sagt jetzt eine **ehrliche** Geschichte über die tatsächlich genutzte Landing-Distance — vorher kannte er nur die Rollout-Distanz in Metern und ignorierte die Bahnlänge komplett.

**Was sich ändert:**

- Score-Formel: `(Touchdown-Distanz + Rollout) / nutzbare Bahnlänge × 100%`
- Heavy-Allowance (-5 pp) für A330/A350/A380/B747/B777/B787/MD11/IL96
- 5 Score-Bänder: `Viel Bahn-Reserve` (100) · `Solider Bremsweg` (80) · `OK — sportlich` (55) · `Lang ausgerollt` (25) · `⚠ Knapp — fast Overrun` (5) · `⚠ Über Bahnende — Overrun-Risiko` (0)
- Sonderfall „vor Displaced-Threshold gelandet": Score wird auf max. 55 PTS gedeckelt + Warning-Pill statt `Viel Reserve` zu zeigen (ehrlich statt schmeichelhaft)
- 6 Skip-Reasons wenn Datenlage unvollständig (`Off-Airport-Landung`, `Bahn-Geometrie nicht vertrauenswürdig`, etc.) — keine erfundenen 100 PTS bei fehlenden Daten

**Beispiel — EK406 A380 in YMML 16, Rollout 584 m:**

- Vorher: 25 PTS „Lang ausgerollt" (weil 584 m > Cessna-Schwelle 800 m und die A380-Kategorie wurde im Pilot-Client nicht beachtet — 8 Monate alter Sync-Bug)
- Jetzt: 100 PTS „Viel Bahn-Reserve" (1.100 m von 3.657 m = 30 %, Heavy-Allowance abgezogen = 25 % → exzellent)

**UI-Erweiterungen:**

- Card zeigt jetzt 3 Werte: gebrauchte Meter · verfügbare Meter · Prozent
- Zusatz-Info-Zeilen unter der Rationale: `davon ~520 m Float vor Aufsetzen` · `Bahn: YMML 16, LDA 3657 m`
- Gelbe Warning-Pill bei Sonderfällen
- Pre-v0.10 PIREPs werden unverändert mit altem Score angezeigt (keine retroaktive Re-Berechnung)

#### 🔧 Update-Modal Hotfix

- Modal hat jetzt `max-height: viewport - padding` → bleibt im sichtbaren Bereich
- Notes-Bereich scrollt separat → Install-Button immer am unteren Rand sichtbar
- Markdown wird leichtgewichtig formatiert: Headings als bold, `**bold**` als bold, `` `code` `` mono, Tabellen werden zu Klartext-Zeilen mit `·` als Separator, `---` als Trenner, Listen als Bullets
- Keine neue NPM-Dependency

## 🇬🇧 English

**v0.10.0 — Runway utilization finally gets honest. Plus: update-modal hotfix (v0.9.x users: please install this version manually — afterwards auto-update works again).**

### ⚠ Important for all v0.9.x users

The update modal in v0.9.0/v0.9.1/v0.9.2 had two bugs combined:

1. Long release notes (like v0.9.2's Discord-RPC + GlitchTip rollout) **overflowed the viewport** — the install button sat below the fold and was unreachable.
2. Raw markdown was rendered un-parsed (`### Tracker`, table pipes, `**bold**`).

**Consequence:** if you're on v0.9.x, **don't** click the in-app modal's "Install". Download `AeroACARS_x64-setup.exe` directly from [GitHub Releases](https://github.com/MANFahrer-GF/AeroACARS/releases/latest) once and install over. From v0.10.0 onwards the auto-update modal works normally again (scrollable, markdown rendered, buttons always visible).

Settings + landing history are preserved across the over-install.

### What's new

#### 🛬 Runway-utilization score rebuilt

The "runway utilization" sub-score now tells an **honest** story about the actually used landing distance — previously it only knew the rollout distance in meters and ignored runway length entirely.

**Changes:**

- Score formula: `(touchdown distance + rollout) / usable runway length × 100%`
- Heavy allowance (-5 pp) for A330/A350/A380/B747/B777/B787/MD11/IL96
- 5 score bands: `Plenty of margin` (100) · `Good stop` (80) · `OK, sporty` (55) · `Long rollout` (25) · `⚠ Marginal — close to overrun` (5) · `⚠ Beyond runway end — overrun risk` (0)
- Special case "touched down before displaced threshold": score capped at 55 PTS + warning pill instead of showing `Plenty of margin` (honest beats flattering)
- 6 skip reasons when data is incomplete (`off-airport landing`, `runway geometry not trusted`, etc.) — no fake 100 PTS on missing data

**Example — EK406 A380 at YMML 16, rollout 584 m:**

- Before: 25 PTS "long rollout" (because 584 m > Cessna threshold 800 m, and the A380 category was ignored in the pilot client — 8-month-old sync bug)
- Now: 100 PTS "plenty of margin" (1100 m of 3657 m = 30 %, minus heavy allowance = 25 % → excellent)

**UI additions:**

- Card now shows 3 values: meters used · meters available · percent
- Extra info lines below the rationale: `~520 m float before touchdown` · `runway: YMML 16, LDA 3657 m`
- Yellow warning pill for special cases
- Pre-v0.10 PIREPs keep their old score (no retroactive re-computation)

#### 🔧 Update-modal hotfix

- Modal now has `max-height: viewport - padding` → stays inside the visible area
- Notes section scrolls independently → install button always anchored at the bottom
- Markdown is lightly formatted: headings as bold, `**bold**` as bold, `` `code` `` mono, tables collapse to plain-text lines with `·` separator, `---` as divider, lists as bullets
- No new NPM dependency

### What pilots need to do

- **v0.9.x → v0.10.0:** install once manually via `AeroACARS_x64-setup.exe` (see note above)
- **from v0.10.0:** auto-update modal works normally again
