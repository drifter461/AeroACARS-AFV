## 🇩🇪 Deutsch

**v0.12.10 — Bugfix: Flugzeugtyp-Erkennung für MSFS-Addons mit rohem ATC-Modell.**

### Was ist neu

#### 🛩️ „Type ?" bei manchen MSFS-Addons behoben

Manche MSFS-Addons — z. B. die **BlackSquare Caravan 208 Professional** — melden ihren `ATC MODEL` als rohen Token-String (`ATCCOM.AC_MODEL C208.0.text`) statt als sauberes ICAO-Kürzel. Dieser rohe String landete **ungereinigt** im Flugzeugtyp-Feld: im Activity-Log stand **„Type ?"**, und der PIREP trug einen unbrauchbaren Flugzeugtyp.

Ab v0.12.10 wird das `ATC MODEL` **schon bei der Erfassung bereinigt**: Aus `ATCCOM.AC_MODEL C208.0.text` wird sauber **`C208`**. Damit zeigt der Client den korrekten Typ an und der PIREP enthält ein gültiges Flugzeug.

### Was Piloten tun müssen

**Nichts** — der Auto-Updater holt v0.12.10 automatisch. Der Fix greift für **neue** Flüge ab v0.12.10; bereits aufgezeichnete Landungen behalten ihren alten Wert.

## 🇬🇧 English

**v0.12.10 — bug fix: aircraft-type detection for MSFS add-ons with a raw ATC model.**

### What's new

#### 🛩️ Fixed "Type ?" for some MSFS add-ons

Some MSFS add-ons — e.g. the **BlackSquare Caravan 208 Professional** — report their `ATC MODEL` as a raw token string (`ATCCOM.AC_MODEL C208.0.text`) instead of a clean ICAO code. That raw string ended up **un-cleaned** in the aircraft-type field: the activity log showed **"Type ?"**, and the PIREP carried an unusable aircraft type.

From v0.12.10 the `ATC MODEL` is **cleaned at capture time**: `ATCCOM.AC_MODEL C208.0.text` now resolves cleanly to **`C208`**. The client shows the correct type and the PIREP carries a valid aircraft.

### What pilots need to do

**Nothing** — the auto-updater fetches v0.12.10 automatically. The fix applies to **new** flights from v0.12.10; already-recorded landings keep their old value.
